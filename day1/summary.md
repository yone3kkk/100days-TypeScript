# Day1: TypeScript環境構築

今日からTypeScriptの勉強を始める
インターンを見据え、モダンな言語を触って使えるようになりたいと思ったからだ。
また、部内活動で共同でwebページを作ったときに、nextやreactを理解していなくて苦労した経歴も影響している。

## 実施内容

1. Node.jsのインストール
	Node.jsはすでにインストールしてあった。
	Node.jsを入れることで、npm（パッケージ管理）とnpx（コマンド実行）が使えるようになる。
	確認として、`node -v` と `npm -v` でバージョンが表示されることをチェックした。

2. VS Codeの準備
	すでにインストールしてあった。

3. TypeScriptのインストール
	ターミナルで `npm install -g typescript` を実行。
	`tsc -v` でコンパイラが使えることを確認。
	gオプションは、グローバルインストールを意味する。

4. プロジェクトの初期化
	作業フォルダを作成し、`tsc --init` を実行して `tsconfig.json` が生成されることを確認。
	この設定ファイルで、コンパイルルールや型チェックの厳しさ（strict など）を管理できる。
    つまり、このファイルは今後の開発スタイルを決める中心ファイルになる。
	TypeScriptは「書いて終わり」ではなく、設定によって品質を段階的に高められるらしい。
