## 1. 今日の到達点

- any 型の特徴と注意点を理解した
- const と let の本質的な違いを説明できる

## 2. 基本型

### string, number, boolean

- 文字列、数値、真偽値を明示して宣言できる
- 型が違う値を代入しようとするとコンパイル時にエラーになる
- テンプレート文字列内の ${...} は値を文字列化して埋め込む

学習例:
- Name, Age, Is Student の出力

---

## 3. 配列型と主要メソッド

### 配列型

- number[] や string[] で要素型を固定できる
- 異なる型を混在させるとエラーになる

### 学んだメソッドと操作

- join: 配列要素を区切り文字で連結し、文字列を返す
- push: 末尾に要素を追加する
- pop: 末尾要素を取り出して返し、配列から削除する
- shift: 先頭要素を取り出して返し、配列から削除する
- 添字アクセス: scores[0] のように要素を読み書きできる

補足:
- enqueue は JavaScript 配列の標準メソッドには存在しない
- キュー操作に近い表現は push + shift

---

## 4. テンプレート文字列

- バッククォートで囲む文字列記法
- ${式} で値や計算結果を埋め込める
- 複数行記述がしやすい
- 文字列連結より可読性が高い

理解ポイント:
- Python の split と join は逆関係
  - split: 文字列 -> 配列
  - join: 配列 -> 文字列

---

## 5. const と let

### let

- 再代入可能

### const

- 再代入不可
- ただし、配列やオブジェクトの中身変更は可能

重要な整理:
- const が固定するのは「変数が参照する先」
- 参照先オブジェクトの内部は変更できる（push, pop など）

つまり:
- ラベルの貼り替え禁止 = const
- ラベルの貼り替え可能 = let

注意:
- const はセキュリティ機能ではない
- 完全な不変性が必要なら readonly, ReadonlyArray, Object.freeze などを使う

---

## 6. オブジェクト型とタプル型

### オブジェクト型

- プロパティ名で意味を表現する
- 読みやすく保守しやすい
- optional プロパティ（?）を使える

例:
- name, age, isStudent? を持つユーザー情報

### タプル型

- 要素数と順番ごとの型を固定する
- [string, number] や [string, number, boolean] のように定義する
- 添字アクセスが基本（0, 1, 2...）

補足:
- タプルは 2 要素に限定されない
- タプルのタプル、タプルの配列、配列の配列はすべて表現可能

---

## 7. any 型

- どんな値でも代入できる型
- 一時的には便利だが、型安全性を下げる
- 多用すると TypeScript を使うメリットが減る

運用指針:
- どうしても型が決まらない場面だけ一時的に使う
- 後で具体的な型に置き換える

---

## 8. 実行方法の理解

### そのまま実行（例: ts-node / tsx）

- 開発時に素早く試せる
- 手軽だが、本番運用は別途設計が必要

### コンパイルして実行（tsc -> node）

- 配布や本番で再現性を取りやすい
- 実行環境との差分管理がしやすい

結論:
- 開発は手軽な実行方式
- CI / 本番は型チェックとビルドを明示する運用が一般的
