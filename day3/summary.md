# Day3: 関数と型の基礎整理

day3では、TypeScriptにおける関数の型付けを重点的に学習した。
C言語での関数定義の感覚を活かしつつ、TypeScriptならではの書き方（アロー関数、optional引数、デフォルト引数）を実装で確認した。

## 実施内容

1. 関数の引数と戻り値の型定義
   - `function greet(name: string): void` のように、引数型と戻り値型を明示する書き方を確認。
   - `function add(a: number, b: number): number` で数値を返す関数を実装。

2. 配列を受け取る関数
   - `number[]` を引数に取り、合計値を返す関数を実装。
   - 空配列を渡した場合の挙動も実行して確認。

3. optional引数とデフォルト引数
   - `isStudent?: boolean` のようなoptional引数を実装。
   - `city: string = "Tokyo"` のようなデフォルト引数を実装。
   - 引数省略時の出力変化を確認。

4. 関数の書き方の比較
   - 関数式（`function`）
   - アロー関数（ブロック形式）
   - アロー関数（1行省略形式）
   - 関数変数への型注釈（`(a: number, b: number) => number`）

5. 実装課題
   - 2つの数値の加算関数
   - 文字列を受け取って出力する`void`関数
   - 名前と年齢（optional）を表示する関数
   - 配列の平均値を返す関数

## 学んだこと

- TypeScriptでは、関数の型を明示することで実行前に不整合を検出できる。
- `void` は「値を返さない」ことを明確に表現できる。
- optional引数を使うと柔軟性が上がる一方、`undefined`の可能性を意識した分岐が必要。
- デフォルト引数は呼び出し側の記述量を減らし、関数の使い勝手を上げられる。
- アロー関数は短く書けるが、型注釈の位置に慣れが必要。

## AIによるレビュー注意点

- 平均値計算は、空配列時に`sum / numbers.length`が`NaN`になる可能性があるため、境界条件の扱いを明確化したい。
- optional引数とデフォルト引数を同時に使う場合、引数順序と省略ルールを整理して使う。
